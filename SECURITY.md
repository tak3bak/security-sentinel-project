## Reporting a Vulnerability

If you discover a potential security vulnerability in this project, please report it via the [GitHub Security Advisory](https://github.com/tak3bak/security-sentinel-project/security/advisories/new) tab. 

When submitting a report, please include the following information to assist our team in triaging the issue:

* **Vulnerability Type:** (e.g., Remote Code Execution, Authentication Bypass, Broken Access Control)
* **Affected Version:** The version of the project where you identified the vulnerability.
* **Steps to Reproduce:** A clear, step-by-step description to demonstrate the issue.
* **Impact:** A summary of how the vulnerability could be exploited and the potential impact on the system.

### What to expect
Once a report is submitted, the core security team will:
1. **Acknowledge** receipt of the report within 48 hours.
2. **Triage** the issue to determine its severity and validity.
3. **Collaborate** with the reporter on a fix and a timeline for public disclosure (if applicable).

We request that you do not publicly disclose the vulnerability until the team has had reasonable time to investigate and patch the issue.
